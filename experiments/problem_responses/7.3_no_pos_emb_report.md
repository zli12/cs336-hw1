# 7.3 NoPE vs RoPE Ablation Report (`no_pos_emb`)

## Setup

- Architecture: TinyStories baseline (`vocab=10000, ctx=256, d_model=512, num_layers=4, num_heads=16, d_ff=1344, RMSNorm pre-norm, SwiGLU`).
- Modification: passed `--no-rope` to `scripts/train_lm.py`, which builds `CausalMultiHeadSelfAttention(use_rope=False)`. RoPE is *not* applied to Q or K; the only positional information available to the model is the implicit causal mask (i.e. "you cannot attend to future positions").
- Training: `bs=128`, `max_steps=3000`, fixed `lr=2e-3`, AdamW (`betas=(0.9, 0.95), eps=1e-8, wd=0.1`), grad clip at norm 1.0.
- Compared against the baseline `lr-sweep-bs128-fixed-lr2e-3.csv` (3000-step LR-tuned run with RoPE, **val loss 1.585**).

## Result

| Run | LR | Final train loss | **Final val loss** | Perplexity |
|---|---:|---:|---:|---:|
| RoPE baseline | 2e-3 | 1.59 | **1.585** | 4.88 |
| **NoPE**      | 2e-3 | 1.67 | **1.667** | 5.30 |

CSV: `experiments/logs/ablation-nope-fixed-lr2e-3.csv`. Plot: `7.3_no_pos_emb.svg`.

NoPE is **+0.082 val loss (+8.6% perplexity)** worse than RoPE at the same compute, with no instability.

## What we observed

NoPE trains stably and tracks the RoPE baseline's overall shape closely. Looking at the validation curves over the 3000 steps:

| step | RoPE val | NoPE val | gap |
|---:|---:|---:|---:|
| 200  | 4.05 | 4.10 | +0.05 |
| 600  | 2.51 | 2.58 | +0.07 |
| 1000 | 2.13 | 2.21 | +0.08 |
| 2000 | 1.78 | 1.85 | +0.07 |
| 3000 | **1.585** | **1.667** | **+0.08** |

The gap opens early (within the first 200 steps) and stays roughly constant at ~0.07-0.08 nats throughout, which suggests NoPE is paying a fixed cost across the entire training trajectory rather than failing to converge. This is consistent with the Tsai et al. / Kazemnejad et al. observation that decoder-only Transformers *can* infer position information from the causal mask alone, but they pay an efficiency cost relative to having position information injected explicitly.

## Discussion

1. **Causal masking gives some position information for free.** A position-`i` token can attend to positions `0..i` but not `i+1..L-1`. The pattern of which positions appear in a token's receptive field is itself positional information, and a sufficiently expressive attention head can learn to count or to distinguish "earliest", "previous", "current". Our 4-layer model with 16 heads has enough capacity to do this, which is why NoPE doesn't collapse - it just does worse.
2. **RoPE is much more sample-efficient than NoPE.** Adding rotation-based phases to Q and K explicitly bakes in relative-position-aware inner products, so the attention weights have a hard-coded inductive bias for "tokens that are 1, 2, 3, ... positions apart". The model doesn't have to learn this from scratch. The +0.08 nats gap at 3000 steps reflects exactly this: NoPE is wasting capacity / steps relearning what RoPE provides analytically.
3. **The gap is bounded but stubborn at this context length.** TinyStories has a context length of 256, which is short enough that NoPE's "infer position from mask" strategy works reasonably well. We would expect the NoPE gap to grow at longer contexts, where keeping track of relative position by counting becomes much harder. RoPE's relative-position parameterization extrapolates much more gracefully.

## Reproduction

```bash
uv run python scripts/train_lm.py \
  --train-data data/tokenized_datasets/tinystories-train.uint16.npy \
  --val-data data/tokenized_datasets/tinystories-dev.uint16.npy \
  --vocab-size 10000 --context-length 256 \
  --d-model 512 --num-layers 4 --num-heads 16 --d-ff 1344 --rope-theta 10000 \
  --batch-size 128 --max-steps 3000 --learning-rate 2e-3 \
  --weight-decay 0.1 --beta1 0.9 --beta2 0.95 --eps 1e-8 --max-grad-norm 1.0 \
  --lr-schedule fixed --device cuda \
  --metrics-csv experiments/logs/ablation-nope-fixed-lr2e-3.csv \
  --no-rope
```
