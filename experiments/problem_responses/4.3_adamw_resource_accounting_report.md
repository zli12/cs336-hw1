# AdamW Resource Accounting

This report answers parts (a) through (d) of the `adamwAccounting` problem.

Architecture assumptions (matching our implementation):

- SwiGLU FFN with three weight matrices (w1, w2, w3), so d_ff = 4 * d_model
- RMSNorm (one scale vector per norm, no bias/shift)
- No bias terms in any linear layer
- RoPE for positions (buffers, not trainable)
- All tensors in float32 (4 bytes per element)

---

## (a) Peak memory decomposition

### Parameters

From the transformer accounting report:

```
params = 2 * vocab_size * d_model
       + num_layers * (4 * d_model² + 3 * d_model * d_ff + 2 * d_model)
       + d_model
```

With d_ff = 4 * d_model, the per-layer weight count simplifies to:

```
4 * d_model² + 12 * d_model² + 2 * d_model = 16 * d_model² + 2 * d_model
```

So:

```
params = 2 * vocab_size * d_model + num_layers * (16 * d_model² + 2 * d_model) + d_model
```

**Memory for parameters:**

```
mem_params = 4 * params  bytes
```

### Gradients

Every trainable parameter has a gradient of the same shape. Gradient memory
equals parameter memory:

```
mem_gradients = 4 * params  bytes
```

### Optimizer state

AdamW stores two moment vectors (m and v) per parameter, each the same shape
as the parameter:

```
mem_optimizer = 2 * 4 * params = 8 * params  bytes
```

### Activations

We need to store intermediate activations for the backward pass. Let
seq_len = context_length and batch = batch_size.

**Per Transformer layer** — we store the input/output of each sub-operation:

1. **RMSNorm 1 input**: batch * seq_len * d_model
2. **RMSNorm 1 output** (= attention input): batch * seq_len * d_model
3. **Q, K, V projections** (3 matrices): 3 * batch * seq_len * d_model
4. **QK^T attention scores**: batch * num_heads * seq_len * seq_len
   = batch * seq_len² * num_heads
5. **Softmax output** (same shape as scores): batch * num_heads * seq_len²
6. **Attention-weighted values** (AV): batch * seq_len * d_model
7. **Output projection result**: batch * seq_len * d_model
8. **RMSNorm 2 input**: batch * seq_len * d_model
9. **RMSNorm 2 output** (= FFN input): batch * seq_len * d_model
10. **FFN w1 output**: batch * seq_len * d_ff
11. **SiLU output** (same shape as w1): batch * seq_len * d_ff
12. **FFN w3 output**: batch * seq_len * d_ff
13. **FFN w2 output** (= FFN result): batch * seq_len * d_model

Per-layer activation element count:

```
act_per_layer = batch * seq_len * (8 * d_model + 3 * d_ff) + 2 * batch * num_heads * seq_len²
```

With d_ff = 4 * d_model:

```
act_per_layer = batch * seq_len * (8 * d_model + 12 * d_model) + 2 * batch * num_heads * seq_len²
              = batch * seq_len * 20 * d_model + 2 * batch * num_heads * seq_len²
```

**Outside the layers:**

14. **Final RMSNorm input**: batch * seq_len * d_model
15. **Final RMSNorm output**: batch * seq_len * d_model
16. **LM head logits**: batch * seq_len * vocab_size
17. **Cross-entropy on logits** (softmax probs): batch * seq_len * vocab_size

Outside-layer activation element count:

```
act_outside = batch * seq_len * (2 * d_model + 2 * vocab_size)
```

**Total activation elements:**

```
act_total = num_layers * (batch * seq_len * 20 * d_model + 2 * batch * num_heads * seq_len²)
          + batch * seq_len * (2 * d_model + 2 * vocab_size)
```

**Activation memory:**

```
mem_activations = 4 * act_total  bytes
```

### Total peak memory

```
mem_total = mem_params + mem_gradients + mem_optimizer + mem_activations
          = 16 * params + 4 * act_total
```

Expanding fully:

```
mem_total = 16 * [2 * vocab_size * d_model + num_layers * (16 * d_model² + 2 * d_model) + d_model]
          + 4 * [num_layers * (batch * seq_len * 20 * d_model + 2 * batch * num_heads * seq_len²)
                + batch * seq_len * (2 * d_model + 2 * vocab_size)]
```

## (b) GPT-2 XL numerical instantiation

GPT-2 XL: vocab_size=50257, seq_len=1024, num_layers=48, d_model=1600,
num_heads=25, d_ff=6400.

### Parameter count (from previous report)

```
params = 2,127,057,600
```

### Non-activation memory (independent of batch_size)

```
mem_params     = 4 * 2,127,057,600   =  8,508,230,400 bytes
mem_gradients  = 4 * 2,127,057,600   =  8,508,230,400 bytes
mem_optimizer  = 8 * 2,127,057,600   = 17,016,460,800 bytes
                                      ----------------
                               total = 34,032,921,600 bytes ≈ 34.03 GB
```

### Activation memory (depends on batch_size)

Per-layer activations (element count):

```
batch * 1024 * 20 * 1600 + 2 * batch * 25 * 1024²
= batch * 32,768,000 + batch * 52,428,800
= batch * 85,196,800
```

All 48 layers:

```
48 * batch * 85,196,800 = batch * 4,089,446,400
```

Outside layers:

```
batch * 1024 * (2 * 1600 + 2 * 50257)
= batch * 1024 * 103,714
= batch * 106,203,136
```

Total activation elements:

```
batch * (4,089,446,400 + 106,203,136) = batch * 4,195,649,536
```

Activation memory:

```
mem_activations = 4 * batch * 4,195,649,536 = batch * 16,782,598,144 bytes
```

### Total

```
mem_total = 16,782,598,144 * batch_size + 34,032,921,600  bytes
```

In the form **a * batch_size + b**:

```
a ≈ 16.78 GB
b ≈ 34.03 GB
```

### Maximum batch size for 80 GB

```
80,000,000,000 ≥ 16,782,598,144 * batch_size + 34,032,921,600
batch_size ≤ (80,000,000,000 - 34,032,921,600) / 16,782,598,144
batch_size ≤ 45,967,078,400 / 16,782,598,144
batch_size ≤ 2.74
```

**Maximum batch_size = 2** (must be an integer).

## (c) FLOPs for one AdamW step

Each AdamW step performs the following per-parameter element-wise operations:

1. `m = beta1 * m + (1 - beta1) * g` — 3 FLOPs per element (multiply, multiply, add)
2. `v = beta2 * v + (1 - beta2) * g²` — 4 FLOPs per element (square, multiply, multiply, add)
3. `alpha_t = alpha * sqrt(1 - beta2^t) / (1 - beta1^t)` — scalar, negligible
4. `theta = theta - alpha_t * m / (sqrt(v) + eps)` — 4 FLOPs per element (sqrt, add, divide, multiply+subtract)
5. `theta = theta - alpha * lambda * theta` — 2 FLOPs per element (multiply, subtract)

Total: about **13 FLOPs per parameter element** per step.

```
flops_adamw = 13 * params
```

For GPT-2 XL:

```
flops_adamw = 13 * 2,127,057,600 ≈ 2.77 × 10^10
```

This is negligible compared to the forward/backward pass FLOPs (~10^13),
which is why optimizer overhead is rarely a bottleneck for compute.

## (d) Training time estimate for GPT-2 XL

### Setup

- 400K training steps, batch_size = 1024, seq_len = 1024
- Forward pass FLOPs (from previous report): 4.513 × 10^12
- Backward pass ≈ 2x forward: 9.026 × 10^12
- Total per-example FLOPs (fwd + bwd): 13.539 × 10^12
- Per-step FLOPs = batch_size * per-example = 1024 * 13.539 × 10^12 = 1.387 × 10^16
- Total training FLOPs = 400,000 * 1.387 × 10^16 = 5.547 × 10^21

### Effective throughput

- A100 peak: 19.5 × 10^12 FLOP/s (float32)
- At 50% MFU: 9.75 × 10^12 FLOP/s

### Wall-clock time

```
time = 5.547 × 10^21 / 9.75 × 10^12
     = 5.690 × 10^8 seconds
     = 6,586 days
     ≈ 18 years
```

**It would take approximately 6,586 days (~18 years) on a single A100.**

This illustrates why large models are trained on hundreds or thousands of GPUs
in parallel — a single accelerator simply cannot complete the job in a
reasonable timeframe.
