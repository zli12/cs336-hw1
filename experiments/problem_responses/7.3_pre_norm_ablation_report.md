# 7.3 Pre-norm vs Post-norm Ablation Report (`pre_norm_ablation`)

## Setup

- Architecture: TinyStories baseline (`vocab=10000, ctx=256, d_model=512, num_layers=4, num_heads=16, d_ff=1344, RoPE θ=10000, RMSNorm, SwiGLU`).
- Modification: `--post-norm` flips each Transformer block from
  - **Pre-norm** (baseline): `x ← x + attn(RMSNorm(x))`, then `x ← x + ffn(RMSNorm(x))`
  - **Post-norm** (this run): `x ← RMSNorm(x + attn(x))`, then `x ← RMSNorm(x + ffn(x))`
  Same parameter count, same dimensions, only the placement of the `RMSNorm` calls changes.
- Training: `bs=128`, `max_steps=3000`, fixed `lr=2e-3`, AdamW (`betas=(0.9, 0.95), eps=1e-8, wd=0.1`), grad clip at norm 1.0.
- Compared against the baseline `lr-sweep-bs128-fixed-lr2e-3.csv` (3000-step LR-tuned run, **val loss 1.585**).

## Result

| Run | LR | Final train loss | Final val loss |
|---|---:|---:|---:|
| Pre-norm baseline | 2e-3 | 1.59 | **1.585** |
| **Post-norm**     | 2e-3 | 5.83 | **5.809** |

CSV: `experiments/logs/ablation-postnorm-fixed-lr2e-3.csv`. Plot: `7.3_pre_norm_ablation.svg`.

## What we observed

Post-norm at the **same `lr=2e-3` that works for pre-norm fails to learn**. Loss profile:

| step | val loss |
|---:|---:|
| 200  | 5.823 |
| 400  | 5.767 |
| 600  | 5.700 |
| 800  | 5.681 |
| 1000 | 5.807 |
| 1500 | 5.752 |
| 2000 | 5.797 |
| 3000 | **5.809** |

Train loss starts at ~9.3 (random for vocab=10k, `ln 10000 ≈ 9.2`), drops to ~5.7 in the first 600 steps, then *plateaus and oscillates* between 5.6 and 5.9 for the remaining 2400 steps. There is no NaN / divergence, just a complete inability to make further progress. The model has essentially memorized a unigram-like distribution and cannot learn the long-range dependencies pre-norm reaches in the same step budget.

By contrast the pre-norm baseline goes from 9.3 → 4.0 by step 200 → ~2.0 by step 1000 → 1.585 by step 3000.

## Discussion

This is the now-canonical result that motivates "pre-norm" as the consensus modern Transformer block:

1. **Gradient signal at depth.** In post-norm, the residual `x + sublayer(x)` is *re-normalized* before being passed on. That means the residual stream's magnitude is reset every block, and gradients from later layers must propagate back through *every* RMSNorm to reach early layers. The Jacobian of RMSNorm scales by `1 / RMS(input)`, and this attenuation compounds over depth, leading to vanishing gradients in early layers. In pre-norm, the residual `x` carries an unmodified copy of the previous layer's output, so the gradient has a clean identity-like path back through the network regardless of how many blocks deep we are.
2. **Optimization is harder; LR has to come down or warmup is needed.** Original Transformer / GPT papers needed long LR warmup precisely because post-norm is fragile at higher LRs. With our `lr=2e-3` (which is the well-tuned LR for pre-norm and close to the pre-norm "edge of stability"), post-norm cannot make stable updates to early-layer parameters and gets stuck. Modern recipes (and ours here) sidestep this entirely by adopting pre-norm.
3. **Lots of training would help, but pre-norm is just better at fixed compute.** With a much smaller LR + warmup, post-norm models *can* train to comparable losses, but they need significantly more steps and tuning. At equal compute, pre-norm wins by a very large margin (val 1.585 vs 5.81 here).

## Reproduction

```bash
uv run python scripts/train_lm.py \
  --train-data data/tokenized_datasets/tinystories-train.uint16.npy \
  --val-data data/tokenized_datasets/tinystories-dev.uint16.npy \
  --vocab-size 10000 --context-length 256 \
  --d-model 512 --num-layers 4 --num-heads 16 --d-ff 1344 --rope-theta 10000 \
  --batch-size 128 --max-steps 3000 --learning-rate 2e-3 \
  --weight-decay 0.1 --beta1 0.9 --beta2 0.95 --eps 1e-8 --max-grad-norm 1.0 \
  --lr-schedule fixed --device cuda \
  --metrics-csv experiments/logs/ablation-postnorm-fixed-lr2e-3.csv \
  --post-norm
```
