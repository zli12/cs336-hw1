# Student Guide: Transformer Resource Accounting

This guide walks you through FLOPs and memory accounting for a decoder-only
Transformer language model. By the end you will be able to count every major
matrix multiply in a forward pass, convert each one to FLOPs, and reason about
how compute shifts as you change model size or context length.

The guide is tailored to the architecture implemented in this homework:

| Component | Detail |
|---|---|
| Positional encoding | RoPE (no trainable positional embeddings) |
| Normalization | RMSNorm (one learnable scale vector per norm, no bias/shift) |
| FFN variant | SwiGLU — three weight matrices (w1, w2, w3) |
| Bias terms | None in any linear layer |
| LM head | Separate `Linear(d_model, vocab_size)`, not weight-tied |

---

## 1  The FLOPs Rule

Almost all floating-point work in a Transformer happens in matrix multiplies.
Everything else (norms, activations, softmax, masks) is negligible by
comparison, so we only need one rule:

> **Rule.** If A is (m x n) and B is (n x p), the product AB costs **2mnp FLOPs**.

Why 2mnp? Each of the m*p output entries requires a dot product of length n,
which is n multiplies + n adds = 2n ops. Total: 2n * m * p = 2mnp.

---

## 2  Architecture Roadmap

A forward pass flows through these stages:

```
tokens --> Token Embedding (lookup, no matmul)
       --> Transformer Block x num_layers
              |-- RMSNorm
              |-- Causal Multi-Head Self-Attention
              |      Q proj   (matmul 1)
              |      K proj   (matmul 2)
              |      V proj   (matmul 3)
              |      QK^T     (matmul 4)
              |      A @ V    (matmul 5)
              |      O proj   (matmul 6)
              |-- + residual
              |-- RMSNorm
              |-- SwiGLU FFN
              |      w1       (matmul 7)
              |      w3       (matmul 8)
              |      w2       (matmul 9)
              |-- + residual
       --> Final RMSNorm
       --> LM Head             (matmul 10)
       --> logits
```

There are **9 matrix multiplies per layer** and **1 at the end** (the LM head).

## 3  Part (a) — Counting Parameters & Memory

### Step 1: List every learnable tensor

GPT-2 XL configuration:

| Name | Value |
|---|---|
| vocab_size | 50,257 |
| seq_len (context_length) | 1,024 |
| num_layers | 48 |
| d_model | 1,600 |
| d_ff | 6,400 |

**Token embedding** — one matrix of shape (vocab_size x d_model):

```
vocab_size * d_model  =  50,257 * 1,600  =  80,411,200
```

**Per Transformer layer** (multiply result by num_layers):

| Tensor | Shape | Param count |
|---|---|---|
| Q proj weight | d_model x d_model | d_model² |
| K proj weight | d_model x d_model | d_model² |
| V proj weight | d_model x d_model | d_model² |
| O proj weight | d_model x d_model | d_model² |
| FFN w1 weight | d_model x d_ff | d_model * d_ff |
| FFN w2 weight | d_ff x d_model | d_ff * d_model |
| FFN w3 weight | d_model x d_ff | d_model * d_ff |
| RMSNorm 1 scale | d_model | d_model |
| RMSNorm 2 scale | d_model | d_model |

Per-layer total = **4 * d_model² + 3 * d_model * d_ff + 2 * d_model**

**Final RMSNorm** — scale vector of size d_model.

**LM head** — weight matrix of shape (d_model x vocab_size):

```
d_model * vocab_size  =  1,600 * 50,257  =  80,411,200
```

### Step 2: Sum it all up

```
Total params = 2 * vocab_size * d_model                                     (embedding + LM head)
             + num_layers * (4 * d_model² + 3 * d_model * d_ff + 2 * d_model)   (layers)
             + d_model                                                       (final RMSNorm)
```

Plugging in GPT-2 XL numbers:

```
Embedding + LM head:  2 * 50,257 * 1,600        = 160,822,400
Per layer:             4*(1,600)^2 + 3*1,600*6,400 + 2*1,600
                     = 10,240,000 + 30,720,000 + 3,200
                     = 40,963,200
All layers:            48 * 40,963,200            = 1,966,233,600
Final RMSNorm:         1,600                      = 1,600

TOTAL = 160,822,400 + 1,966,233,600 + 1,600
      = 2,127,057,600 parameters
```

### Step 3: Memory

Each FP32 parameter is 4 bytes:

```
2,127,057,600 * 4  =  8,508,230,400 bytes  ≈  8.51 GB
```

**Answer (a):** GPT-2 XL has about **2.13 billion trainable parameters**. In
FP32 the weights alone need roughly **8.51 GB** of memory.

## 4  Part (b) — Forward-Pass FLOPs

We process an input of shape (seq_len x d_model) through the model. Below we
list every matrix multiply, its dimensions, and its FLOPs cost.

### 4.1  Per-layer matrix multiplies

Let X be the (seq_len x d_model) input to a layer.

> **SwiGLU note:** The FFN has three matrices, not two. w1 and w3 both take
> the same input X (they run in parallel). Their outputs are combined via
> `H = SiLU(X @ W1) * (X @ W3)`, and then w2 projects back down:
> `output = H @ W2`. That is why the execution order is w1, w3, w2 — and why
> w2's input is labeled H (the gated intermediate) rather than X.

| # | Operation | Left dim | Right dim | FLOPs (2mnp) |
|---|---|---|---|---|
| 1 | Q projection: X @ W_Q | seq_len x d_model | d_model x d_model | 2 * seq_len * d_model² |
| 2 | K projection: X @ W_K | seq_len x d_model | d_model x d_model | 2 * seq_len * d_model² |
| 3 | V projection: X @ W_V | seq_len x d_model | d_model x d_model | 2 * seq_len * d_model² |
| 4 | Attention scores: Q @ K^T | seq_len x d_model | d_model x seq_len | 2 * seq_len² * d_model |
| 5 | Attention-weighted values: A @ V | seq_len x seq_len | seq_len x d_model | 2 * seq_len² * d_model |
| 6 | Output projection: O @ W_O | seq_len x d_model | d_model x d_model | 2 * seq_len * d_model² |
| 7 | FFN w1: X @ W1 | seq_len x d_model | d_model x d_ff | 2 * seq_len * d_model * d_ff |
| 8 | FFN w3: X @ W3 | seq_len x d_model | d_model x d_ff | 2 * seq_len * d_model * d_ff |
| 9 | FFN w2: H @ W2 (H = gated intermediate) | seq_len x d_ff | d_ff x d_model | 2 * seq_len * d_ff * d_model |

**Per-layer total:**

```
Attention projections (Q,K,V,O):    4 * 2 * seq_len * d_model²    = 8 * seq_len * d_model²
Attention matmuls (QK^T, AV):       2 * 2 * seq_len² * d_model    = 4 * seq_len² * d_model
FFN matmuls (w1, w2, w3):           3 * 2 * seq_len * d_model * d_ff = 6 * seq_len * d_model * d_ff
```

### 4.2  Model-level matmul

| # | Operation | Left dim | Right dim | FLOPs |
|---|---|---|---|---|
| 10 | LM head: X @ W_vocab | seq_len x d_model | d_model x vocab_size | 2 * seq_len * d_model * vocab_size |

### 4.3  Grand total

```
Total FLOPs = num_layers * (8 * seq_len * d_model²
                          + 4 * seq_len² * d_model
                          + 6 * seq_len * d_model * d_ff)
            + 2 * seq_len * d_model * vocab_size
```

### 4.4  GPT-2 XL numbers

```
Per layer:
  Attention projections:  8 * 1024 * 1600^2         =  20,971,520,000
  Attention matmuls:      4 * 1024^2 * 1600          =   6,710,886,400
  FFN matmuls:            6 * 1024 * 1600 * 6400     =  62,914,560,000
  Layer subtotal:                                    =  90,596,966,400

All 48 layers:            48 * 90,596,966,400        = 4,348,654,387,200

LM head:                  2 * 1024 * 1600 * 50257    =   164,682,137,600

TOTAL                                               = 4,513,336,524,800
                                                     ≈ 4.51 × 10^12 FLOPs
```

## 5  Part (c) — Where Do the FLOPs Go?

Break the total into component groups to see what dominates:

| Component | FLOPs | % of total |
|---|---|---|
| FFN (w1 + w2 + w3) | 3,019,898,880,000 | **66.9%** |
| Attention projections (Q,K,V,O) | 1,006,632,960,000 | **22.3%** |
| Attention matmuls (QK^T + AV) | 322,122,547,200 | **7.1%** |
| LM head | 164,682,137,600 | **3.6%** |

**Answer (c):** At seq_len = 1,024 the **FFN** is the biggest compute
consumer by far (about two-thirds of all FLOPs). The quadratic attention
matmuls are modest here because seq_len = 1,024 is not large enough for the
seq_len² term to dominate.

## 6  Part (d) — Scaling Across GPT-2 Variants

All variants share vocab_size = 50,257 and seq_len = 1,024. The FFN inner
dimension follows d_ff = 4 * d_model for every variant.

| Variant | num_layers | d_model | d_ff |
|---|---|---|---|
| Small | 12 | 768 | 3,072 |
| Medium | 24 | 1,024 | 4,096 |
| Large | 36 | 1,280 | 5,120 |
| XL | 48 | 1,600 | 6,400 |

### How to compute each variant

Use the same formula for each:

```
Total = num_layers * (8 * seq_len * d_model²
                    + 4 * seq_len² * d_model
                    + 6 * seq_len * d_model * d_ff)
      + 2 * seq_len * d_model * vocab_size
```

Just plug in the variant's num_layers, d_model, and d_ff.

### Results table

| Component | Small | Medium | Large | XL |
|---|---|---|---|---|
| FFN | 49.8% | 59.9% | 64.2% | 66.9% |
| Attn projections | 16.6% | 20.0% | 21.4% | 22.3% |
| Attn matmuls (QK^T, AV) | 11.1% | 10.0% | 8.6% | 7.1% |
| LM head | 22.6% | 10.2% | 5.8% | 3.6% |
| **Total FLOPs** | 3.50 x 10^11 | 1.03 x 10^12 | 2.26 x 10^12 | 4.51 x 10^12 |

### How to read the trend

Think about how each term scales with d_model:

- **FFN and Attention projections** scale as d_model² (or d_model * d_ff,
  and d_ff = 4 * d_model). When d_model doubles, these terms roughly 4x.
- **Attention matmuls** scale as seq_len² * d_model. When only d_model (not
  seq_len) grows, these go up linearly in d_model — much slower than the
  d_model² terms above.
- **LM head** scales as d_model * vocab_size. vocab_size is fixed, so it
  grows linearly in d_model.

**Answer (d):** As models get wider and deeper, the d_model²-scaling terms
(FFN and attention projections) take a larger share of total FLOPs. The LM
head and attention matmuls, which only grow linearly in d_model, shrink
proportionally. The LM head drops from ~23% in Small to ~4% in XL.

## 7  Part (e) — Extending Context Length to 16,384

Take GPT-2 XL (num_layers=48, d_model=1600, d_ff=6400) and set
seq_len = 16,384 instead of 1,024.

### What changes

Every FLOPs term contains seq_len, so everything grows — but the key is
*how fast*:

- Terms with **seq_len** (projections, FFN, LM head): grow **16x**
  (since 16,384 / 1,024 = 16).
- Terms with **seq_len²** (QK^T, AV): grow **256x** (since 16^2 = 256).

### New numbers

```
Per layer:
  Attention projections:  8 * 16384 * 1600^2        =   335,544,320,000
  Attention matmuls:      4 * 16384^2 * 1600         = 1,717,986,918,400
  FFN matmuls:            6 * 16384 * 1600 * 6400    = 1,006,632,960,000
  Layer subtotal:                                    = 3,060,164,198,400

All 48 layers:            48 * 3,060,164,198,400     = 146,887,881,523,200

LM head:                  2 * 16384 * 1600 * 50257   =   2,634,914,201,600

TOTAL                                               = 149,522,795,724,800
                                                     ≈ 1.50 × 10^14 FLOPs
```

This is about **33x** the FLOPs at seq_len = 1,024 (not just 16x), because
the quadratic attention terms blew up by 256x.

### New component shares

| Component | seq_len = 1,024 | seq_len = 16,384 |
|---|---|---|
| FFN | 66.9% | **32.3%** |
| Attn projections | 22.3% | **10.8%** |
| Attn matmuls (QK^T, AV) | 7.1% | **55.2%** |
| LM head | 3.6% | **1.8%** |

**Answer (e):** Increasing context length to 16,384 raises forward-pass
FLOPs by about **33x** (to ~1.50 x 10^14) because the attention matmul
terms grow quadratically with seq_len. The compute bottleneck flips: FFN was
dominant at seq_len = 1,024 but attention score/value matmuls now consume
over half of all FLOPs. This is why long-context research focuses heavily on
efficient attention mechanisms.

## Quick-Reference Cheat Sheet

### Parameter formula

```
params = 2 * vocab_size * d_model
       + num_layers * (4 * d_model² + 3 * d_model * d_ff + 2 * d_model)
       + d_model
```

### FLOPs formula (one forward pass)

```
flops = num_layers * (8 * seq_len * d_model²
                    + 4 * seq_len² * d_model
                    + 6 * seq_len * d_model * d_ff)
      + 2 * seq_len * d_model * vocab_size
```

### Scaling intuitions

| When you increase... | Linear terms grow by | Quadratic terms grow by |
|---|---|---|
| d_model (model width) | ~d_model | ~d_model² |
| num_layers (depth) | ~num_layers | ~num_layers |
| seq_len (context length) | ~seq_len | ~seq_len² |

The crossover point — where attention matmuls overtake FFN — happens when
seq_len becomes comparable to d_ff. For GPT-2 XL that is around
seq_len ~ 6,400.
