# 7.3 Layer-Norm Ablation Report (`layer_norm_ablation`)

## Setup

- Architecture: TinyStories baseline (`vocab=10000, ctx=256, d_model=512, num_layers=4, num_heads=16, d_ff=1344, RoPE θ=10000, pre-norm, SwiGLU`).
- Modification: passed `--no-rms-norm` to `scripts/train_lm.py`, which replaces both `RMSNorm` layers in every Transformer block with `nn.Identity()` (residual stream is *not* normalized at all). Embeddings, RoPE, attention, and FFN are unchanged.
- Training: `bs=128`, `max_steps=3000`, fixed LR (no warmup/decay), AdamW (`betas=(0.9, 0.95), eps=1e-8, wd=0.1`), grad clip at norm 1.0.
- Compared against the baseline `lr-sweep-bs128-fixed-lr2e-3.csv` (3000-step LR-tuned run, RMSNorm enabled, **val loss 1.585**).

## Runs

| Run | LR | Final train loss | Final val loss | Behavior |
|---|---:|---:|---:|---|
| Baseline (RMSNorm)               | 2e-3 | 1.59 | **1.585** | stable |
| No-RMSNorm                       | 2e-3 | NaN  | NaN       | catastrophic divergence |
| No-RMSNorm                       | 5e-4 | 1.69 | **1.684** | stable, ~6.3% worse perplexity |

CSVs: `experiments/logs/ablation-nonorm-fixed-lr2e-3.csv`, `experiments/logs/ablation-nonorm-fixed-lr5e-4.csv`.

## What happens at the previously optimal LR (`lr=2e-3`)?

The model **diverges immediately** when RMSNorm is removed. Train loss progression:

| step | train loss |
|---:|---:|
| 1   | 16.2 |
| 50  | 7.56 |
| 100 | **4.6e3** |
| 150 | **1.62e6** |
| 200 | **1.02e7** |
| 200 (val) | **4.9e13** |
| 250 | 1.64e13 |
| 350 | 5.11e13 |
| 550 | 3.22e22 |
| ≥ ~1400 | NaN forever |

Loss explodes by ~5 orders of magnitude in the first 100 steps, then keeps blowing up until activations overflow to NaN/Inf. Gradient clipping at 1.0 keeps the optimizer step bounded, but the *forward pass* itself produces unbounded activations because each block adds an unnormalized residual back into the stream. After a few SwiGLU/attention layers, magnitudes compound multiplicatively across blocks and tokens.

## Can we recover stability with a lower LR?

**Yes - at `lr=5e-4` (the same LR that worked for `bs=8` in the bs-sweep), the no-RMSNorm model trains stably and reaches val loss 1.684**, only ~0.10 above the RMSNorm baseline. The training loss decreases monotonically throughout, with no spikes. We did not need to fall back to `lr=1e-4`.

## Commentary on the impact of RMSNorm

RMSNorm has **two distinct, complementary effects**:

1. **Forward-pass scale control.** In a pre-norm Transformer the residual stream `x` accumulates contributions from every block: `x ← x + attn(norm(x))`, then `x ← x + ffn(norm(x))`. Without RMSNorm, those contributions stack with no rescaling, so block outputs see an input whose RMS grows roughly linearly with depth. SwiGLU is bilinear in the hidden activation, so it amplifies the growth. With 4 layers this is enough to push activations into FP-overflow territory in a few hundred steps. RMSNorm prevents this by re-anchoring the input distribution to each block to RMS=1.

2. **Effective learning rate / gradient scaling.** Because RMSNorm normalizes the residual *before* each sublayer, gradients flowing back through `attn` and `ffn` see well-conditioned inputs. Removing it means each sublayer's effective LR is amplified by the (growing) input norm, which is why the *same* `lr=2e-3` that was healthy with RMSNorm is now too large by a wide margin. We had to drop the LR by 4x to land in a stable basin, and even then we leave performance on the table.

These two together explain why even at the recovered LR (5e-4) the no-RMSNorm model is measurably worse: stability was bought by undertraining (4x lower LR for the same step budget), so it sees less effective optimization. **RMSNorm is not just a "stability hack" - it lets you use a larger LR per parameter, which is what actually closes the gap to the baseline.**

## Reproduction

```bash
# Baseline (already exists)
# experiments/logs/lr-sweep-bs128-fixed-lr2e-3.csv

# Divergent run
uv run python scripts/train_lm.py \
  --train-data data/tokenized_datasets/tinystories-train.uint16.npy \
  --val-data data/tokenized_datasets/tinystories-dev.uint16.npy \
  --vocab-size 10000 --context-length 256 \
  --d-model 512 --num-layers 4 --num-heads 16 --d-ff 1344 --rope-theta 10000 \
  --batch-size 128 --max-steps 3000 --learning-rate 2e-3 \
  --weight-decay 0.1 --beta1 0.9 --beta2 0.95 --eps 1e-8 --max-grad-norm 1.0 \
  --lr-schedule fixed --device cuda \
  --metrics-csv experiments/logs/ablation-nonorm-fixed-lr2e-3.csv \
  --no-rms-norm

# Stable lower-LR run
uv run python scripts/train_lm.py ... --learning-rate 5e-4 \
  --metrics-csv experiments/logs/ablation-nonorm-fixed-lr5e-4.csv \
  --no-rms-norm
```
