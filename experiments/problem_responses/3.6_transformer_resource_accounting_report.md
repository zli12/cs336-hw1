# Transformer LM Resource Accounting

This report answers parts (a) through (e) using the Transformer architecture implemented in this homework:
- token embedding + `num_layers` pre-norm Transformer blocks + final RMSNorm + LM head
- RoPE is used for attention positions (RoPE caches are buffers, not trainable parameters)
- no bias terms in linear layers

Throughout, we use:
- `V` = vocab size
- `S` = sequence length (context length at full input)
- `d_m` = `d_model`
- `L` = number of layers
- `d_ff` = FFN inner dimension

FLOPs rule used: for `A (m x n)` times `B (n x p)`, cost is `2mnp` FLOPs.

---

## (a) GPT-2 XL parameters and model memory

Given:
- `V = 50,257`, `S = 1,024`, `L = 48`, `d_m = 1,600`, `d_ff = 6,400`

### Trainable parameters

Per model:

1. Token embedding: `V * d_m`
2. LM head: `V * d_m`
3. Per layer:
   - Attention projections `Q, K, V, O`: `4 * d_m * d_m`
   - FFN (`w1, w2, w3`): `3 * d_m * d_ff`
   - RMSNorm gains (`ln1`, `ln2`): `2 * d_m`
4. Final RMSNorm gain: `d_m`

Total:

`Params = 2 * V * d_m + L * (4 * d_m² + 3 * d_m * d_ff + 2 * d_m) + d_m`

Numerically:
- `2 * V * d_m = 160,822,400`
- `L(...) = 1,966,233,600`
- `+ d_m = 1,600`
- **Total = 2,127,057,600 parameters**

Memory for fp32 weights:
- `2,127,057,600 * 4 bytes = 8,508,230,400 bytes`
- about **8.51 GB** (decimal) or **7.92 GiB** (binary)

**Deliverable (a), concise:** GPT-2 XL in this architecture has **2,127,057,600 trainable parameters**. Stored in fp32, loading only the weights needs about **8.51 GB** (about **7.92 GiB**) of memory.

## (b) Matrix multiplies in one forward pass (GPT-2 XL, S = 1,024)

Below is the matrix-multiply list for a single example (batch size 1), with costs.

### Per Transformer layer

1. **Q projection**: `(S x d_m) @ (d_m x d_m)` -> `2 * S * d_m * d_m`
2. **K projection**: `(S x d_m) @ (d_m x d_m)` -> `2 * S * d_m * d_m`
3. **V projection**: `(S x d_m) @ (d_m x d_m)` -> `2 * S * d_m * d_m`
4. **Attention score matmul (`QKᵀ`)** across all heads -> `2 * S * S * d_m`
5. **Attention-value matmul (`softmax(QKᵀ)V`)** across all heads -> `2 * S * S * d_m`
6. **Attention output projection**: `(S x d_m) @ (d_m x d_m)` -> `2 * S * d_m * d_m`
7. **FFN w1**: `(S x d_m) @ (d_m x d_ff)` -> `2 * S * d_m * d_ff`
8. **FFN w3**: `(S x d_m) @ (d_m x d_ff)` -> `2 * S * d_m * d_ff`
9. **FFN w2**: `(S x d_ff) @ (d_ff x d_m)` -> `2 * S * d_ff * d_m`

Per-layer total:

`8 * S * d_m * d_m + 4 * S * S * d_m + 6 * S * d_m * d_ff`

### Whole model extras

10. **Final LM head**: `(S x d_m) @ (d_m x V)` -> `2 * S * d_m * V`

### Numeric FLOPs (XL)

Per layer:
- `8 * S * d_m * d_m = 20,971,520,000`
- `4 * S * S * d_m = 6,710,886,400`
- `6 * S * d_m * d_ff = 62,914,560,000`
- Layer subtotal = `90,596,966,400`

All layers (`L=48`):
- `48 * 90,596,966,400 = 4,348,654,387,200`

LM head:
- `2 * S * d_m * V = 164,682,137,600`

**Total FLOPs (forward pass):**
- **`4,513,336,524,800` FLOPs** (about `4.51e12`)

## (c) Which parts dominate FLOPs for GPT-2 XL?

For `S=1,024`, the FFN matrix multiplies dominate:
- FFN matmuls: `3,019,898,880,000` FLOPs (**66.9%** of total)
- attention projections: `1,006,632,960,000` (**22.3%**)
- attention `QKᵀ` + `AV`: `322,122,547,200` (**7.1%**)
- final LM head: `164,682,137,600` (**3.6%**)

**Deliverable (c), concise:** The FFN multiplies use the most compute by far (about two-thirds of total FLOPs at `S=1024`). Attention score/value matmuls are smaller here because their `S²` term is not yet dominant at this context length.

## (d) Small / Medium / Large FLOPs proportion breakdown

Assumptions for GPT-2 family:
- same `V = 50,257`, `S = 1,024`
- small: `L=12`, `d_m=768`, `d_ff=3072`
- medium: `L=24`, `d_m=1024`, `d_ff=4096`
- large: `L=36`, `d_m=1280`, `d_ff=5120`

We group components into:
- **Attention projections** (`Q,K,V,O` linear layers)
- **Attention score/value matmuls** (`QKᵀ` and `AV`)
- **FFN matmuls** (`w1,w3,w2`)
- **Final LM head**

### GPT-2 Small (12L, d_m=768)
- Total FLOPs: `349,630,365,696`
- FFN: `173,946,175,488` -> **49.75%**
- Attention projections: `57,982,058,496` -> **16.58%**
- Attention score/value: `38,654,705,664` -> **11.06%**
- LM head: `79,047,426,048` -> **22.61%**

### GPT-2 Medium (24L, d_m=1024)
- Total FLOPs: `1,033,109,504,000`
- FFN: `618,475,290,624` -> **59.86%**
- Attention projections: `206,158,430,208` -> **19.96%**
- Attention score/value: `103,079,215,104` -> **9.98%**
- LM head: `105,396,568,064` -> **10.20%**

### GPT-2 Large (36L, d_m=1280)
- Total FLOPs: `2,257,754,521,600`
- FFN: `1,449,551,462,400` -> **64.20%**
- Attention projections: `483,183,820,800` -> **21.40%**
- Attention score/value: `193,273,528,320` -> **8.56%**
- LM head: `131,745,710,080` -> **5.84%**

**Deliverable (d), concise trend:** As model size grows (larger `L` and `d_m`, fixed `S`), the `O(L * S * d_m²)` terms (especially FFN, then projection layers) take a larger share of FLOPs. The LM head (`O(S * d_m * V)`) and attention score/value matmuls (`O(L * S² * d_m)`) become proportionally smaller.

## (e) GPT-2 XL with context length 16,384

Now set `S = 16,384` (16x longer than 1,024), keeping XL `L=48, d_m=1600, d_ff=6400`.

New total FLOPs:
- **`149,522,795,724,800`** (about `1.50e14`)

Compared to `S=1,024` total `4.513e12`, this is about:
- **33.1x higher FLOPs**, not just 16x, because attention score/value matmuls scale as `S²`.

Component shares at `S=16,384`:
- Attention score/value matmuls: **55.15%** (now dominant)
- FFN: **32.31%**
- Attention projections: **10.77%**
- LM head: **1.76%**

**Deliverable (e), concise:** Increasing XL context length to 16,384 raises forward FLOPs by about **33x** (to `1.495e14`) because quadratic-attention terms grow with `S²`. The dominant compute shifts from FFN at short context to attention score/value matmuls at long context.
