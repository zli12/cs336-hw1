# 7.1 Experiment Log (TinyStories LM)

## Logging Infrastructure

Training/evaluation metrics are logged by `scripts/train_lm.py` in three places:

- **Console logs** print `step`, `train_loss` or `val_loss`, throughput, and `elapsed_s`.
- **Weights & Biases (optional)** logs:
  - `train/loss`
  - `val/loss`
  - `perf/tokens_per_sec`
  - `time/elapsed_sec`
  - all aligned to `step`.
- **CSV artifact (optional)** via `--metrics-csv <path>` writes:
  - `step,wallclock_sec,split,loss,tokens_per_sec`
  - one row per logging event (`split=train` or `split=val`).

This supports plotting loss curves with respect to both gradient steps and wallclock time.

## Command Template

```bash
uv run python scripts/train_lm.py \
  --train-data <train_tokens.npy> \
  --val-data <val_tokens.npy> \
  --vocab-size <vocab_size> \
  --context-length <context_length> \
  --batch-size <batch_size> \
  --d-model <d_model> \
  --num-layers <num_layers> \
  --num-heads <num_heads> \
  --d-ff <d_ff> \
  --max-steps <max_steps> \
  --learning-rate <lr> \
  --log-every <log_every> \
  --val-every <val_every> \
  --val-batches <val_batches> \
  --checkpoint-path experiments/checkpoints/<run_name>.pt \
  --metrics-csv experiments/logs/<run_name>.csv \
  --wandb --wandb-project cs336-basics --wandb-run-name <run_name>
```

## How To Produce Curves

1. Run training with `--metrics-csv`.
2. Plot `loss` vs `step` grouped by `split`.
3. Plot `loss` vs `wallclock_sec` grouped by `split`.
4. For each ablation/hyperparameter run, copy summary metrics into the table above.
