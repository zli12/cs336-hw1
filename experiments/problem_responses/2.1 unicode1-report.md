# 2.1 Problem (unicode1): Understanding Unicode

## Problem Statement

**Problem (unicode1): Understanding Unicode (1 point)**  
(a) What Unicode character does `chr(0)` return?  
Deliverable: A one-sentence response.  
  
(b) How does this character’s string representation (`__repr__()`) differ from its printed representation?  
Deliverable: A one-sentence response.  
  
(c) What happens when this character occurs in text? It may be helpful to play around with the following in your Python interpreter and see if it matches your expectations:  
`>>> chr(0)`  
`>>> print(chr(0))`  
`>>> "this is a test" + chr(0) + "string"`  
`>>> print("this is a test" + chr(0) + "string")`  
Deliverable: A one-sentence response.

## Answers

## (a)
**Question:** What Unicode character does `chr(0)` return?  
**Answer:** `chr(0)` returns the Unicode NULL control character (code point U+0000).

## (b)
**Question:** How does this character’s `__repr__()` differ from its printed representation?  
**Answer:** Its `__repr__()` is an escaped literal like `'\x00'`, while `print()` emits the actual control character, which is typically invisible.

## (c)
**Question:** What happens when this character occurs in text?  
**Answer:** When this character appears inside text, it is still part of the string contents but usually does not display as a visible symbol in terminal output.
